### 1. INTRODUCCIÓN. <a name="id1"></a>

QuickCart es una plataforma innovadora para la creación rápida y sencilla de páginas web, así como la gestión de productos, carritos y pagos, pero centrándose en el aspecto de diseño y desarrollo web.


El principal objetivo de este documento es detallar el desarrollo y funcionalidad de QuickCart, una aplicación que revoluciona la forma en que se crean y gestionan páginas web de forma intuitiva y eficiente.
Comenzamos definiendo el contexto y la lógica de la plataforma, centrándonos en la necesidad de una herramienta que simplifique la creación de sitios comerciales y ayude a facilitar la gestión de productos y el comercio en línea.
Nuestro principal objetivo es proporcionar una visión completa del proyecto, desde su establecimiento hasta la implementación de sus principales hitos.
En cuanto a la gama QuickCart, destacan características como la creación de sitios web con una interfaz sencilla e intuitiva, gestión de catálogo e inventario de productos, integración del carrito de compras y opciones de pago seguro.


También analizamos aspectos del diseño y desarrollo de aplicaciones, detallando la arquitectura, la tecnología utilizada, las consideraciones de diseño y la usabilidad para garantizar una experiencia fluida y atractiva para los usuarios. Se destaca funcionalidades clave como la personalización de páginas.


Para lanzar QuickCart se desarrolló un plan estratégico que incluía estrategias de marketing para atraer y retener usuarios.
Los resultados se evalúan a través de métricas de éxito, como las tasas de conversión, y se recopilan comentarios de los usuarios para futuras mejoras y actualizaciones de la plataforma.

### 2. ORIGEN, CONTEXTUALIZACIÓN Y JUSTIFICACIÓN DEL PROYECTO. <a name="id2"></a>

QuickCart nace de la necesidad creciente de los emprendedores y dueños de negocios de contar con una plataforma ágil y efectiva para la creación y gestión de sus páginas web. Observamos que muchos negocios enfrentan dificultades al intentar desarrollar su presencia en línea de manera eficiente, buscando recursos que les permitan mejorar su visibilidad y sus ventas. Al mismo tiempo, identificamos que existen numerosas herramientas complejas en el mercado que dificultan este proceso para muchos usuarios.

Nuestro objetivo es abordar esta problemática mediante una plataforma sencilla y accesible. QuickCart permite a los usuarios crear sus páginas web de forma rápida y sin complicaciones, ofreciendo una amplia gama de herramientas para gestionar productos, carritos de compra y pagos de manera efectiva. Además, QuickCart fomenta la interacción y colaboración entre los usuarios, permitiendo que compartan recursos y experiencias para optimizar sus ventas en línea.

En resumen, QuickCart surge como respuesta a la necesidad de los emprendedores y dueños de negocios de contar con una herramienta intuitiva y eficiente para la creación y gestión de sus páginas web. La plataforma busca simplificar el proceso de desarrollo web y potenciar las ventas en línea de manera efectiva y accesible para todos los usuarios.

### 3. OBJETIVO GENERAL DEL PROYECTO. <a name="id3"></a>

El objetivo principal de QuickCart es crear una plataforma eficiente y fácil de usar para crear y administrar sitios web comerciales.
Nuestro objetivo era desarrollar una herramienta que simplifique el proceso de diseño y desarrollo web, permitiendo a los usuarios crear sitios web de forma rápida y eficiente, con funciones avanzadas de gestión de productos y opciones seguras de compra y pago.


QuickCart se esfuerza por proporcionar un entorno virtual que fomente la creación de sitios web personalizados, optimice para dispositivos móviles y proporcione una experiencia de usuario atractiva y fluida. Nuestro objetivo es facilitar a los usuarios la creación y gestión de sus sitios web, mejorando así su presencia online y sus oportunidades de ventas.


En QuickCart nos enfocamos en desarrollar una plataforma que permita a los usuarios crear y administrar sus sitios web de manera efectiva y eficiente, mejorando así el éxito en línea y simplificando el proceso de desarrollo web para todos los usuarios.


### 4. DESCRIPCIÓN GENERAL DEL PROYECTO (ALCANCE). <a name="id4"></a>

QuickCart es una aplicación diseñada para simplificar la creación de páginas web comerciales mediante una interfaz intuitiva y eficiente. La aplicación se estructura en varias partes clave que permiten a los usuarios acceder a sus funciones y características específicas.

La aplicación utiliza una arquitectura cliente-servidor para garantizar un rendimiento óptimo. Los usuarios interactúan a través de la interfaz de cliente en sus dispositivos, mientras que el servidor se encarga del procesamiento de solicitudes, la gestión de datos y la funcionalidad necesaria para la creación y gestión de las páginas.

En términos de seguridad, QuickCart implementa un sólido sistema de autenticación para proteger la privacidad de los usuarios. Esto incluye procesos de inicio de sesión seguros mediante la encriptación de la contraseña.

Una vez autenticados, los usuarios tienen acceso a diversas funciones clave de QuickCart:

1. Creación de páginas web: Los usuarios pueden crear sus propias páginas web utilizando plantillas prediseñadas y herramientas de personalización.
2. Gestión de productos y catálogo: QuickCart permite a los usuarios gestionar su inventario de productos, agregar descripciones, imágenes y precios de manera fácil y eficiente.
3. Carritos de compra y pagos: La aplicación facilita la integración de carritos de compra para que los clientes puedan realizar compras en línea de forma segura y conveniente.
4. Diseño responsive: QuickCart garantiza que las páginas web creadas sean totalmente responsivas, adaptándose a diferentes dispositivos y pantallas.

En términos de tecnologías, QuickCart se desarrolla Angular Ionic y Django RestApi, lo que garantiza un desarrollo optimizado para dispositivos webs y movil. Además, se implementan medidas de seguridad como el jwt_token y la encriptación de  para proteger la privacidad de los usuarios y sus transacciones en línea.

QuickCart ofrece una solución completa para la creación y gestión de páginas web comerciales, con un enfoque en la simplicidad, la eficiencia y la seguridad para los usuarios que buscan establecer su presencia en línea de manera efectiva.

### 9. POLÍTICA DE SEGUIMIENTO, EVALUACIÓN Y CONTROL DE CALIDAD. <a name="id9"></a>

Este apartado tiene como objetivo garantizar que el proyecto se desarrolle de acuerdo con los estándares establecidos y cumpla con los objetivos planteados. Para lograr esto, se llevarán a cabo las siguientes acciones:

1. Establecimiento de métricas de calidad: Estas métricas pueden incluir la eficiencia del código, la usabilidad de la aplicación, la ausencia de errores críticos, entre otros aspectos relevantes.

2. Revisión continua del código: Se establecerán revisiones periódicas del código desarrollado por el equipo, con el fin de identificar posibles problemas de calidad, como errores, código no optimizado o malas prácticas de programación. Estas revisiones permitirán corregir las desviaciones de calidad de manera temprana y mejorar la robustez del proyecto.

3. Utilización de herramientas de control: Se emplearán herramientas de control o testeo, para identificar posibles problemas y garantizar la integridad del código y la funcionalidad de la aplicación.

4. Implementación de pruebas: Se llevarán a cabo pruebas del código para verificar la integridad de este. Y así prevenir errores.

5. Realización de auditorías: Se llevarán a cabo auditorías periódicas para evaluar el progreso del proyecto en relación con lo establecido en el cronograma y los requisitos impuestos. Estas auditorías permitirán identificar problemas, corregir desviaciones y garantizar el correcto desarrollo del proyecto.

6. Seguimiento del cumplimiento de requisitos: Se realizará una serie de seguimientos para asegurar que los requisitos y especificaciones del proyecto se estén cumpliendo.

7. Mejora continua:  Refiriéndose a la corrección de problemas de calidad, así como la implementación de acciones preventivas y correctivas.


### 10.  CLAUSULAS. <a name="id10"></a>
Algunas cláusulas que podrían incluirse en un contrato pueden ser:

- Retraso en la entrega: En caso de que se incumpla con los plazos acordados para la entrega del proyecto, se establecerá una cláusula de penalización por retraso. Esto puede implicar una reducción en el monto total a pagar o la bajada de nota  por cada día de retraso.

- Pagos: Se establecerá un cronograma de pagos detallado que indique las fechas y montos específicos a pagar a medida que se cumplan ciertos hitos o etapas del proyecto.

- Control de calidad: Se establecerá un proceso de control de calidad para garantizar que el producto final cumpla con los estándares y requisitos acordados.

- Correcciones y mejoras: Existe un período de corrección y ajuste posterior a la entrega del proyecto, durante el se realizan las correcciones necesarias y abordar cualquier problema o defecto identificado.

- Propiedad intelectual: Se establecerán cláusulas que definan la propiedad intelectual y los derechos de propiedad sobre el software desarrollado. 

- Confidencialidad: Se establecerá una cláusula de confidencialidad que proteja la información y los datos confidenciales compartidos durante el proyecto.
